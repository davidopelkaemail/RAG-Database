1. What are Embeddings?

Embeddings are numerical representations of words, sentences, or concepts. They convert text into vectors (lists of numbers) that capture semantic meaning. Words with similar meanings have similar vectors!

Key Insight: In embedding space, "king - man + woman ≈ queen" because the vector relationships capture semantic concepts!

2. Attention Mechanism

The attention mechanism allows the model to focus on different parts of the input when processing each word. It answers: "When processing this word, which other words should I pay attention to?"

Why it matters: In "The cat sat on the mat", when processing "it", the model needs to know "it" refers to "cat" - attention weights make this possible!

Self-Attention Formula

Attention(Q, K, V) = softmax(QKT / √dk) × V

Where:
- Q (Query): "What am I looking for?"
- K (Key): "What do I contain?"
- V (Value): "What do I represent?"
- dk: Dimension of keys (for scaling)

4. Transition Layers (Feed-Forward Networks)

After attention, each token passes through a feed-forward network. This is where the model "thinks" about what it learned from attention and transforms it.

Input

[Attention Output]

d_model = 768

→
Layer 1

Linear

768 → 3072

→
Activation

GELU/ReLU

non-linear

→
Layer 2

Linear

3072 → 768

→
Output

[Transformed]

d_model = 768

Why expand then contract? The expansion to 4x size (768 → 3072) allows the model to learn more complex representations. Think of it as "unfolding" the information to process it more thoroughly before "folding" it back.
